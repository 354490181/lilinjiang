hexo博客图片无法显示问题记录

在今天写技术博客的时间，突然发现需要上传一个图片，由于本身对Hexo并不熟悉，故求助于度娘和谷歌。
翻看很多博客最终解决，总结一下操作：
1.在_config.yml中将post_asset_folder设置为true，某些博主说它的这个文件里并没有这个设置，我猜想你可以手动添加，
因为据我了解，这个是开启对应博文资源搜集的文件夹设置功能，打开它具有一定的必要性；

2.安装图片显示插件，划个重点，要在指定的hexo文件夹下安装，安装命令如下（任意一个均可<其实都是安装hexo-asset-image插件>）：
npm install https://github.com/CodeFalling/hexo-asset-image –save 我使用的这个安装成功的
npm install hexo-asset-image –save

3.创建新的博文 hexo new “xxxx”,发现/source/_post目录下有一个与xxxx同名的文件夹，这个是步骤一的作用；

4.将图片放入文件夹中，在博文中使用markdown语法写入图片![随便写](图片路径)；

5.运行如下命令hexo clean -> hexo deploy -g ，刷新博客文章可以看到对应的图片展示出来了。

图片是否能够展示其实在hexo g 这个命令是就可以看出，如果public/index.html文件中存在

证明能够显示成功。

注意：第二部时可能会出现npm安装失败的问题，不妨修改一下下载镜像，如将其修改为淘宝镜像
（npm i -g cnpm –registry=https://registry.npm.taobao.org），在使用npm安装命令时，将npm更改为cnpm即可。
